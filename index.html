Sorry Elisabeth

In develpment since January, Sorry Elisabeth is a story based horror 2d game. You will play as Elisabeth and resolve a couple of enigmas to understand what happened and how to get out of that mysterious house in which you have woken up.

See on Github

  • Interaction system

    • Player can interact with several objects in the room.
    • Different interation's type (examine, open, use, combine...).
    • Player can not interact with certain objects depending on the room lighting.

  • Inventory system

    • Different inventory's size depending on if the player has a handbag or not.
    • Interaction system adapted to the inventory's item.

  • Full point and click experience

    • Handling different actions on the same area with only one user input.

  • Learning pixel art & artistic constraints

    • Configure the engine to respect a proper pixel art style.
    • Player's animation and UI textures done trough the developement.

Tools : Godot, C++, Git, Aesprite

Divided Soul - 12 hours Game Jam

Made in a 3 person team on the theme : "Stronger Together", Divided Soul is a 2d puzzle game in co-op. You play two souls on the same keyboard which have to merge together to complete a level. 2 levels were maid, and we are now looking for developing the concept.

See on Github

  • Adaptability

    • Produced two playable levels in one night while completely discovering Unity and C#.
    • Include fast brainstorming and team managment.

  • Collision system

    • Level objects detect wether a player is entering or exiting thier body.
    • Interact with the player's physics depending on the object's purpose.

  • Pressure plate system

    • Open a linked door if the right player is on the pressure plate.

  • Scene transition & animation

    • Scene fade transition implemented.
    • Player's animation designed using Aesprite and Unity's animation system.

Tools : Unity, C#, Git, Aesprite

Retro 1010!

Retro 1010! is a Tetris like inspired by the quite famouse game "1010!". You need to place pieces of diferent sizes and forms on a 10 by 10 board, and try to always have enough places to do so. You earn points when placing a piece or supressing a line by fill up an entire row or column. Finished during my free time in high school, I spent a good time developing alone all parts of the game, including the sound system and the texture design.

See on Github

  • Random piece generation

    • Developed several pieces with diferent sizes and forms.
    • Randomly choose to generate 3 pieces every round.

  • Handling player's action

    • The player can choose which piece he want to place on the board with the same inputs as the ones that handle pieces movements and settings interactions.
    • Determine wether the player has lost or not at various states of the game.

  • Keeps track of game's informations

    • Save local best score and display it on every game over screen.

  • Sound system

    • Sound settings integrated.
    • Player can choose to listen to the game music or not, as well to the sound effects.

Tools : Python, Pygame, Git, Aesprite, Audacity

Towa

Towa is a plate game invented by my formation which oppose two players on a board with a predefine number of rounds. Each round, several actions are possible to the player which can increase or decrease its number of pieces. The player's goal is, at the end of the predefine number of rounds, to have more pieces than his oponent on the board.

See on Github

  • Player's actions and consequences

    • Several player's action which have many consequences on the game.
    • Rigorous problems refactoring.

  • Carefull unit testing work

    • Unit tests done for each actions and refactored classes methods.

  • Expert system IA developed in duo

    • First brainstorming phase.
    • Diferent anticipations and reactions to the oponent's actions depending on if we play first or second.

Tools : Java, Git
