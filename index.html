Sorry Elisabeth

In development since January, Sorry Elisabeth is a story-based horror game in 2D. You will play as Elisabeth, a young British girl. Your goal is to resolve puzzles to understand what happened and how to get out of this mysterious house in which you have woken up.

See on Github

  • Player interactions

    • Can interact with several objects in the room.
    • Diferent actions available (examine, open, use, combine...).
    • Player can't interact with certain objects depending on the room's state (lighting...).

  • Inventory management

    • Diferent inventory size depending on the player's equipment (ie if they have a handbag or not)
    • Interaction system changes depending on items in the player's inventory

  • Full point and click experience

    • Handling diferent actions on the same area with only one user input (mouse).

  • Learning pixel art & artistic constraints

    • Setting up the engine to respect a pleasant pixel art style.
    • Character animations and UI textures self-taught troughout the developement.

Tools : Godot, C++, Aesprite

Divided Soul - 12 hours Game Jam

Made in a 3-person team with the theme: "Stronger Together", Divided Soul is a 2D puzzle game in cooperation. You play two split souls on the same keyboard which have to merge together to become complete (and complete the level!). We created 2 levels and are now looking to push forward this one-night idea.

See on Github

  • Adaptability and efficiency

    • Produced 2 playable levels in less than 12 hours while completely discovering Unity and C#.
    • Includes phases of brainstorming and team managment.

  • Players merging

    • Players play on different sides of the same keyboard (like an old-school multiplayer experience!)
    • They need to solve some puzzles and meet in the middle to merge into 1 and go to the next stage

  • Pressure plate system

    • Opening a linked door needs 1 player to stay on a pressure plate.

  • Scene transition & animation

    • Scene fade transitions implemented.
    • Player animations designed using Aesprite and Unity's animation system.

Tools : Unity, C#, Aesprite

Retro 1010!

Retro 1010! is a Tetris-like game, inspired by the quite famous "1010!". You need to place pieces of diferent sizes and shapes on a 10 by 10 board, and try to always have enough places to do so. You earn points when placing a piece, or supressing a line by filling up an entire row or column. I did this game during my free time in high school and spent a good amount of time developing alone all parts of the game, including the sound system and the texture design.

See on Github

  • Random pieces generation

    • Developed several pieces with diferent sizes and shapes.
    • 3 pieces randomly generated every round.

  • Handling player actions

    • The player can choose which piece they want to place on the board.
    • The inputs are the same for handling pieces movements and settings interactions.
    • Determine whether the player has lost or not at various states of the game.

  • Stored state management

    • Saves the local best score and displays it on every game over screen.

  • Sound system

    • Sound settings integrated.
    • Player can choose to listen to the game music or not, as well as to the sound effects.

Tools : Python, Pygame, Aesprite, Audacity

Towa

Towa is a board game developed during my classes which oppose two players on a board with a predefined number of rounds. Each round, the player has multiple actions available which can increase or decrease the number of pieces. The player's goal is, at the end of the predefined number of rounds, to have more pieces than their oponent.

See on Github

  • Player actions and consequences

    • Tha player can choose between various diferent actions (put on the board, activate a tower...).
    • Rigorous problems refactoring (packages, classes, sub-methods...).

  • Unit testing

    • Unit tests done for all class methods.

  • Expert system AI developed in pair

    • Anticipate and react differently depending if the player is player first or second.

Tools : Java
