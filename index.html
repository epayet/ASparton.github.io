Sorry Elisabeth

In develpment since January, Sorry Elisabeth is a story based horror 2d game. You will play as Elisabeth, a british young girl and resolve enigmas to understand what happened and how to get out of this mysterious house in which you have woken up.

See on Github

  • Interaction system

    • Player can interact with several objects in the room.
    • Diferent interation's type (examine, open, use, combine...).
    • Player can't interact with certain objects depending on the room state (lighting...).

  • Inventory system

    • Diferent inventory's size depending on if the player has a handbag or not.
    • Interaction system adapted to the inventory's item.

  • Full point and click experience

    • Handling diferent actions on the same area with only one user input.

  • Learning pixel art & artistic constraints

    • Setting up the engine to respect a pleasant pixel art style.
    • Player's animations and UI textures done by myself trough the developement.

Tools : Godot, C++, Git, Aesprite

Divided Soul - 12 hours Game Jam

Maid in a 3 person team on the theme : "Stronger Together", Divided Soul is a 2d puzzle game in cooperation. You play two souls on the same keyboard which have to merge together to complete a level. 2 levels were maid, and the team is now looking to push forward this one night idea.

See on Github

  • Adaptability

    • Produced two playable levels in less than 12 hours while completely discovering Unity and C#.
    • Include brainstorming's phase and team managment.

  • Players merging

    • When the players collide, merge them and load the next scene

  • Pressure plate system

    • Open a linked door if the right player is on the pressure plate.

  • Scene transition & animation

    • Scene fade transition implemented.
    • Player's animations designed using Aesprite and Unity's animation system.

Tools : Unity, C#, Git, Aesprite

Retro 1010!

Retro 1010! is a Tetris like inspired by the quite famouse game "1010!". You need to place pieces of diferent sizes and shapes on a 10 by 10 board, and try to always have enough places to do so. You earn points when placing a piece, or supressing a line by filling up an entire row or column. Finished during my free time in high school, I spent a good time developing alone all parts of the game, including the sound system and the texture design.

See on Github

  • Random pieces generation

    • Developed several pieces with diferent sizes and shapes.
    • Randomly choose to generate 3 pieces every round.

  • Handling player's action

    • The player can choose which piece he want to place on the board with the same inputs as the ones that handle pieces movements and settings interactions.
    • Determine wether the player has lost or not at various states of the game.

  • Keeps track of game's informations

    • Saves the local best score and displays it on every game over screen.

  • Sound system

    • Sound settings integrated.
    • Player can choose to listen to the game music or not, as well as to the sound effects.

Tools : Python, Pygame, Git, Aesprite, Audacity

Towa

Towa is a plate game invented by my formation which oppose two players on a board with a predefine number of rounds. Each round, several actions are possible to the player which can increase or decrease its number of pieces. The player's goal is, at the end of the predefine number of rounds, to have more pieces than his oponent on the board.

See on Github

  • Player's actions and consequences

    • Tha player can choose between various diferent actions (put on the board, activate a tour...).
    • Rigorous problems refactoring (packages, classes, sub-methods...).

  • Carefull unit testing

    • Unit tests done for each classes methods.

  • Expert system AI developed in duo

    • Diferent anticipations and reactions to the oponent's actions depending on if we play first or second.

Tools : Java, Git
